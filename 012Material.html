    Material for MkDocs

    MkDocs incluye dos temas estándar, MkDocsy ReadTheDocs. Si queremos buscar otros temas, podemos acudir a la wiki comunitaria para explorar la lista de temas disponibles.

    Para mis cuadernos, voy a utilizar Material

    Véase su repositorio GitHub aquí

    Guía rápida

    Tenemos el manual de uso aquí. Podemos instalarlo en nuestro equipo con el comando

    pip install mkdocs-material

    Y activarlo modificando el archivo de configuración del proyecto mkdocs.yml:

    theme:
  name: 'material'

    … y al generar el sitio web, veremos el resultado

    Otra posibilidad es descargarlo directamente de GitHub, acudiendo a su repositorio y pulsando en el botón [clone or download]. Esto resulta útil si queremos hacer muchas modificaciones en el tema para adaptarlo a nuestro gusto. En ese caso, el archivo de configuración será:

    theme:
  name: null
  custom_dir: 'mkdocs-material/material'

    Colores

    Este tema define los colores de acuerdo al juego definido en la biblioteca material de google. Podemos configurarlo de la siguiente forma:

    theme:
  palette:
    primary: 'indigo'
    accent: 'indigo'

    Los nombres de colores son case-insensitive. Los posibles valores son red, pink, purple, deep purple, indigo, blue, light blue, cyan, teal, green, light green, lime, yellow, amber, orange, deep orange, brown, grey, blue grey y white. Los últimos cuatro solo pueden usarse como primary.

    Si hacemos uso de estas opciones de configuración, se añadirá un archivo css automáticamente.

    Tenemos una paleta de colores para experimentar en la guía del tema. Pulsar aquí

    El valore por defecto es: indigo.

    Fuente

    Por defecto la fuente es Roboto y Roboto Mono. Véase referencia. Ambas fuentes forman parte de Google Fonts y podemos cambiarlas por otras, por ejemplo, Ubuntu font family.

    theme:
  font:
    text: 'Ubuntu'
    code: 'Ubuntu Mono'

    Podemos desactivar las fuentes proporcionadas con:

    theme:
  font: false

    Logo

    El icono por defecto es: school. Debe tener una resolución mínima de 128x128, dejando espacio en los costados, y presentando un contraste suficiente con el color de fondo. La forma de establecer otro icono es poniéndolo en la carpeta docs/images, y configurándolo con:

    theme:
  logo: 'images/logo.svg'

    Otra posibilidad es hacer referencia por su nombre a alguno de los definidos en Material Design icon font. Por ejemplo:

    theme:
  logo:
    icon: 'cloud'

    Idiomas

    Podemos especificar el idioma de la página con:

    theme:
  language: 'es'

    Siendo ‘en’ el valor por defecto.

    Para establecer el idioma para las búsquedas escribimos:

    extra:
  search:
    language: 'es, en'

    Favicon

    El valor por defecto es assets/images/favicon.png. Puede cambiarse mediante:

    theme:
  favicon: 'assets/images/favicon.ico'

    Pestañas

    Material contempla la posibilidad de un nivel superior en forma de menú horizontal. Activarlo con:

    theme:
  feature:
    tabs: true

    Enlace a repositorio

    Para añadir un enlace al repositorio de documentación, configurar en mkdocs.yml:

    repo_name: 'squidfunk/mkdocs-material'
repo_url: 'https://github.com/squidfunk/mkdocs-material'

    El nombre del repositorio aparecerá junto al recuadro de búsqueda o en la barra de navefación, dependiendo del tamaño de la ventana.

    Si repo_url apunta a GitHub, BitBucket o GitLab, se mostrará el logo junto al enlace. Para GitHub, también el número de estrellas y forks.

    Si el repositorio se ubica en un entorno privado, el logo puede establecerse poniendo extra.repo_icon como github, gitlab o bitbucket.

    ¿Por que hay un botón editar al inicio de cada artículo?

    Si asignamos repo_url a GitHub o a BitBucket, repo_name incluye GitHub o BitBucket de forma implícita o explícita, un botón de edición aparecerá al inicio de cada artículo. Este es el comportamiento automático que MkDocs implementa. Véase la documentación de MkDocs para más información.

    Enlaces a redes sociales

    Podemos mostrar redes sociales en el pie de página usando la fuente FontAwesome. El parámetro type mostrará el nombre del servicio, y link la URL a enlazar:

    extra:
  social:
    - type: 'github'
      link: 'https://github.com/squidfunk'
    - type: 'twitter'
      link: 'https://twitter.com/squidfunk'
    - type: 'linkedin'
      link: 'https://linkedin.com/in/squidfunk'

    Los enlaces se muestran en ese orden, y type debe coincidir con el nombre definido en FontAwesome. Se añadirá automáticamente fa, por lo que github se convertirá en fa fa-github.

    Google Analytics

    MkDocs facilita la integración con Google Analytics, lo que permite analizar el uso de nuestro repositorio. Podemos activarlo mediante el archivo de configuración: mkdocs.yml:

    google_analytics:
  - 'UA-XXXXXXXX-X'
  - 'auto'

    Disqus

    Material for MkDocs está integrado con Disqus, lo que permite una sección de comentarios. En la configuración tenemos que añadir nuestro nombre:

    extra:
  disqus: 'miNombre'

    La sección de comentarios se insertará en todas las páginas excepto index. En la tabla de contenido se generará un enlace a la sección de comentarios.

    Requerimientos

    Se debe establecer un valor para site_url

    Disqus puede establecerse o desactivarse para determinadas páginas mediante metadatos.

    Arquitectura del tema

    Si queremos hacer modificaciones al tema Material, tendremos que configurar mkdocs.yml como de costumbre:

    theme:
  name: 'material'
  custom_dir: 'tema'

    La estructura del directorio del tema debe ser reflejo de la del tema original. Todo archivo en la capeta apuntada por custom_dir reemplazará al original con el mismo nombre. Además, podemos añadir nuevos ficheros.

    La disposición de Material es la siguiente:

    .
├─ assets/
│  ├─ images/                          # Imagenes e iconos
│  ├─ javascripts/                     # JavaScript
│  └─ stylesheets/                     # Hojas de estilo
├─ partials/
│  ├─ integrations/                    # 3rd-party 
│  ├─ language/                        # Idiomas
│  ├─ footer.html                      # Pie de página
│  ├─ header.html                      # Cabecera
│  ├─ hero.html                        # Hero teaser
│  ├─ language.html                    # Etiquetas
│  ├─ nav-item.html                    # Elemento de la barra
│  ├─ nav.html                         # Barra de navegación
│  ├─ search.html                      # Recuadro de búsqueda
│  ├─ social.html                      # Redes sociales
│  ├─ source.html                      # Repositorio
│  ├─ tabs-item.html                   # Elemento de pestaña
│  ├─ tabs.html                        # Pestañas
│  ├─ toc-item.html                    # Elemento de TOC
│  └─ toc.html                         # Tabla de contenido
├─ 404.html                            # Página 404
├─ base.html                           # Plantilla Base
└─ main.html                           # Página por defecto

    Sobreescribir una sección

    Si por ejemplo queremos cambiar el pie de página, reemplazar footer.html por un archivo partials/footer.html en el directorio apuntado por custom_dir.

    Sobreescribir bloques de plantila

    Podemos crear un nuevo archivo mail.html implementando solo los bloques que queramos personalizar. Se respetará el resto del archivo original.

    Por ejemplo:

    {% extends "base.html" %}

{% block htmltitle %}
  <title>Lorem ipsum dolor sit amet</title>
{% endblock %}

    Material proporciona los siguientes bloques:

    Nombre Incluye:
    analytics Integración con Google Analytics
    content Contenido de la página
    disqus Integración con disqus
    extrahead Bloque vacío para definir etiquetas meta añadidas
    fonts Definición de fuentes
    footer Pie de página con navegación y copyright
    header Barra de cabecera fija
    hero Portada
    htmltitle Etiqueta <title>
    libs Bibliotecas JavaScript
    repo Enlace al repositorio en la barra de cabecera
    scripts Scripts de JavaScript
    source Archivos fuente enlazados
    search_box Recuadro de búsqueda
    site_meta Meta etiquetas en sección <head>
    site_name Nombre del sitio en la cabecera
    site_nav Navegación y tabla de contenidos
    social Enlaces a redes sociales
    styles Hojas de estilo
