    Creando un sitio web

    Una vez que nos hemos dado de alta en GitHub y estamos preparados para crear documentos en esa plataforma, vamos a desarrollar un sitio web experimental a partir de nuestros contenidos.

    Organizar los repositorios

    GitHub Pages es un servicio especial de GitHub que convierte documentos en páginas web. Podemos tener tantos sitios web como queramos, uno por repositorio. Las personas que quieran acceder a nuestra página deben introducir como URL:

    https://nombreUsuario.github.io/nombreProyecto

    sustituyendo nombreUsuario y nombreProyecto por lo que corresponda.

    Podemos crear un sitio genérico a nivel de usuario, accesible mediante la URL:

    https://nombreUsuario.github.io

    Para ello debemos poner la documentación en un repositorio cuyo nombre será “nombreUsuario.github.io”. En mi caso particular, tengo lo siguiente:

    • el nombre de usuario es cuadernosdenotas
    • para la página de portada tengo un repositorio llamado cuadernosdenotas.github.io
    • para cada cuaderno tengo otro repositorio con el nombre que corresponda, accesible mediante la URL cuadernosdenotas.github.io/nombrecuaderno.

    Se puede examinar esta disposición accediendo a mi lista de repositorios.

    Para los ejemplos que vienen a continuación, he creado un usuario “de pruebas” llamado flxc.

    Nuestra primera página

    Comenzamos por crear un nuevo repositorio llamado github.com/flxc/prueba, con un archivo README.md.

    Vamos a crear un sitio web a partir de ese repositorio. Cada página será un archivo HTML. Lo habitual es que la página principal sea un texto llamado index.html. Dejaremos README.md como documentación.

    Con la lista de archivos del repositorio a la vista, pulsamos sobre el botón “Create new file” e introducimos lo siguiente:

    <!DOCTYPE html>
<html>
<head>
</head>
<body>

<h1>Mi primera página web</h1>

<p>Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed eiusmod
 tempor incidunt ut labore et dolore magna aliqua. Ut enim ad minim 
 veniam, quis nostrud exercitation ullamco laboris nisi ut aliquid ex ea 
 commodi consequat.</p>

<p>Quis aute iure reprehenderit in voluptate velit esse cillum dolore eu 
fugiat nulla pariatur. Excepteur sint obcaecat cupiditat non proident, 
sunt in culpa qui officia deserunt mollit anim id est laborum.</p>

</body>
</html>

    y lo guardamos como index.html.

    Lo que vemos es el típico archivo de texto en formato html, que es el estándar básico de las páginas web. El contenido de los ficheros es una mezcla de texto y etiquetas < >. Estas últimas representan instrucciones para el navegador.

    Ahora tendremos dos archivos, uno de texto en formato markdown llamado readme.md con una descripción del proyecto, y el otro en formato html llamado index.html, que contiene el texto de nuestra primera página web. Vamos a decirle a GitHub que queremos convertir todo esto en un sitio web.

    En la página principal del repositorio, pulsamos en el enlace [settings].

    Esto nos da acceso a la página de configuración del repositorio. Bajamos hasta el apartado GitHub Pages, y en el subapartado Source tenemos un botón para seleccionar la rama/carpeta donde se ubican los documentos. Cambiamos none por master branch y pulsamos en el botón [Save] para activar el generador de páginas.

    Se pone en marcha el generador del sitio web. Podemos ver el estado del mismo entrando en esta página settings, apartado GitHub Pages, y viendo el mensaje que se muestra, que puede ser:

    • GitHub Pages is currently disabled. La generación automática de páginas web está desactivada (podemos desactivarla seleccionando de nuevo none como rama del repositorio).

    • Your site is ready to be published at https://nombreusuario.github.io/. Se está llevando a cabo la compilación. Nótese que GitHub Pages puede tener unos límites, y si hemos rebasado el número máximo de compilaciones por hora, este mensaje se mantendrá hasta que se desbloquee el generador.

    • Your site is published at https://nombreusuario.github.io/ (sobre fondo verde). Se ha ejecutado la compilación con éxito.

    • Your site is having problems building: texto de error (sobre fondo amarillo). Se ha producido un fallo de compilación.

    Nota:

    La compilación y generación de páginas se mantiene activada de forma permanente y actuará cada vez que modifiquemos cualquier archivo del repositorio, refrescando nuestro sitio web. Si vamos a introducir muchas modificaciones, conviene desactivar el servicio temporalmente seleccionando la opción None en la lista Source de la página settings.

    Cuando se muestre el mensaje:

    Your site is published at https://usuario.github.io/repositorio/

    podemos pulsar sobre ese enlace para examinar nuestra nueva página web. El resultado es el siguiente:

    El formato html

    Las etiquetas HTML obedecen a un estándar denominado Hipertext Markup Language, coordinado por un consorcio mundial llamado w3c.

    Veamos una breve explicación de las etiquetas utilizadas en este primer ejemplo:

    • La primera etiqueta es <!DOCTYPE html>. Le dice al navegador que lo que viene a continuación es un documento que se ajusta a ese estándar.

    • La etiqueta <html> delimita el comienzo del documento. Se corresponde con la etiqueta final </html> que sirve como etiqueta de cierre.

    • La etiqueta <head> delimita el comienzo de la cabecera del documento. Se corresponde con la etiqueta </head> que sirve de etiqueta de cierre de esa sección.

      La cabecera del documento se utiliza para incorporar definiciones e información acerca de la página web. Por ahora, hemos dejado esta sección vacía.

    • La etiqueta <body> y su correspondiente etiqueta de cierre </body> delimitan el “cuerpo” del documento, es decir, el texto y elementos visibles.

    • Las etiquetas <p> y </p> delimitan párrafos.

    • Si queremos un párrafo de título, lo delimitaremos con etiquetas <h1> ... </h1>. En un documento estructurado en capítulos, subcapítulos y apartados, se admiten hasta seis niveles de líneas de cabeceras, siendo las de nivel 1 las que se muestran más resaltadas. Se representa con etiquetas que van de <h1> hasta <h6> y sus correspondientes etiquetas de cierre de </h1> hasta </h6>.

    Nota:

    HTML es un formato complejo. Otras etiquetas permiten representar enlaces, imágenes, tablas, listas, y un largo etcétera.

    Nota:

    En el formato HTML, las líneas en blanco, saltos de línea, sangrado y espaciado extra en general, será ignorado por el navegador en el momento de visualizar la página.

    Estilos

    Cambiando el tipo de letra, los colores, alineación, espaciado, etc. crearemos páginas más vistosas. Podemos establecer unas reglas de estilo y ponerlas en un archivo especial llamado “hoja de estilos”.

    Supongamos que, por ejemplo, queremos lo siguiente:

    • cuerpo de la página, representado por la etiqueta body: poner un fondo de color beige suave.

    • cabeceras de nivel uno, representadas por etiquetas <h1>: texto blanco sobre fondo verde. Alineación del texto: centrado.

    • resto de párrafos delimitados por etiquetas <p>: texto en color verde oscuro.

    Para cada tipo de elemento creamos una colección de definiciones de atributos, donde cada uno de ellos se escribe en formato atributo:valor. Cada atributo se delimita con un punto y coma final, y el conjunto de definiciones se delimita entre llaves. Por ejemplo, para las cabeceras, escribiríamos:

    h1 {
    background-color: green;
    color:white;
    text-align:center;
}

    Este formato se denomina css (cascading style sheets), y su sintaxis, nombres y posibles valores de atributos siguen las normas establecidas por el w3c.

    Vamos a crear un archivo llamado estilos.css con el siguiente contenido:

    body {background-color: cornsilk;}

h1   {background-color: green;
      color:white;
      text-align:center;
     }

p { color: darkgreen; }

    Nota:

    Al igual que sucede con el formato HTML, las líneas en blanco, saltos de línea, sangrado y espaciado extra en general, será ignorado por el navegador en el momento de visualizar la página.

    Hemos asignado nombres de colores de acuerdo a los estándares css. Véase la referencia aquí. También podríamos usar los códigos de cada color, precedidos por un carácter #:

    body {background-color: #FFF8DC;}

    En el archivo index.html tenemos que indicar que queremos enlazar con esa hoja de estilos. Para ello utilizaremos la etiqueta <link>, que colocaremos en las definiciones de la sección <head>. Aprovecharemos para incluir una definición del título que queremos visualizar en la barra superior de la ventana del navegador:

    <!DOCTYPE html>
<html>
<head>

    <title>Mi primera página web</title>

    <link rel="stylesheet" href="estilos.css">

</head>
<body>

<h1>Mi primera página web</h1>

<p>Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed eiusmod
 tempor incidunt ut labore et dolore magna aliqua. Ut enim ad minim 
 veniam, quis nostrud exercitation ullamco laboris nisi ut aliquid ex ea 
 commodi consequat.</p>

<p>Quis aute iure reprehenderit in voluptate velit esse cillum dolore eu 
fugiat nulla pariatur. Excepteur sint obcaecat cupiditat non proident, 
sunt in culpa qui officia deserunt mollit anim id est laborum.</p>

</body>
</html>

    Toda etiqueta html sirve para delimitar un elemento de la página web, y los atributos del elemento se pueden establecer en la etiqueta de apertura usando el formato:

    <etiqueta atributo1="valor 1" atributo2="valor 2"> texto </etiqueta>

    El texto entre la etiqueta de apertura y la de cierre es el elemento delimitado por esas etiquetas. Cuando una etiqueta es un elemento en si misma, como sucede con <link>, no habrá etiqueta de cierre y se dice que tenemos un elemento “vacío”.

    Los atributos se escriben siempre en la etiqueta de apertura, y se separan con espacios en blanco. En este caso utilizamos el atributo rel para establecer que el archivo enlazado es una “hoja de estilos” y el atributo href para hacer referencia al nombre del archivo a enlazar:

    <link rel="stylesheet" href="estilos.css">

    Tras modificar index.html y crear estilos.css, al refrescarse la página web veremos que ha cambiado el aspecto:

    La propiedad style

    Lo anterior sirve para establecer las reglas de estilo para todos los elementos del mismo tipo (cabeceras, párrafos, etc.). Pero… ¿como podemos cambiar el estilo de un elemento individual sin afectar a los otros? Escribiendo las reglas de estilo directamente en la etiqueta de apertura del elemento. Utilizaremos el atributo style:

    <h1 style=" ...reglas de estilo... ">Mi primera página web</h1>

    Las reglas de estilo se escriben en formato:

    atributo1:valor1; atributo1:valor1; atributo1:valor1;

    con lo que en nuestro ejemplo tendremos:

    <h1 style="background-color: green;
           color:white;
           text-align:center;">Mi primera página web</h1>

    Recuérdese que el espaciado y saltos de línea son irrelevantes en archivos html.

    Otra forma de asignar un estilo a un elemento individual es asignarle un nombre mediante el atributo id:

    <h1 id="cabecera">Mi primera página web</h1>

    Esto nos permite hacer referencia a ese elemento en la hoja de estilos:

    body {background-color: cornsilk;}

#cabecera   {background-color: green;
             color:white;
             text-align:center;
            }

p { color: darkgreen; }

    En formato css, para hacer referencia a un elemento por su id, este debe escribirse con un prefijo #.

    ¿Como podemos asignar un estilo especial a varios elementos sin que afecte a todos los del mismo tipo? Supongamos que queremos resaltar en color rojo los párrafos pares y no hacerlo con los impares. La solución es agrupar los elementos afectados, y asignarles un nombre de “clase”:

    <p class="pares">texto del párrafo</p>

    Ahota tenemos varios párrafos declarados como miembros de la clase “pares”. Tenemos que definir el atributo “class” en todos ellos.

    En la hoja de estilos, las clases van precedidas por un punto “.”

    .pares { color=red;}

    Enlaces a otras páginas

    En nuestro sitio web podemos crear tantos archivos HTML como queramos, cada uno representando una página, aunque la que se visualizará automáticamente será siempre index.html. Para visualizar el resto de las páginas, tenemos que insertar enlaces en los textos. Para ello incluimos una etiqueta en formato:

    <a>texto del enlace</a>.

    Por ejemplo:

    <h1>Mi primera página web</h1>

<h2>Indice de páginas</h2>

<p>Capítulo 1: <a>pulsar aquí</a></p>
<p>Capítulo 2: <a>pulsar aquí</a></p>
<p>Capítulo 3: <a>pulsar aquí</a></p>

    Y para cada línea visualizaremos el texto:

    Capítulo x: pulsar aquí

    Véase que el texto del enlace forma parte del párrafo. Pero nos falta indicar para cada enlace cual es el archivo a enlazar. Para este propósito, usaremos el atributo href de la etiqueta <a>:

    <h1>Indice de páginas</h1>

<p>Capítulo 1: <a href="capitulo1.html">pulsar quí</a></p>
<p>Capítulo 2: <a href="capitulo2.html">pulsar quí</a></p>
<p>Capítulo 3: <a href="capitulo3.html">pulsar quí</a></p>

    Seguidamente tendremos que crear los archivos enlazados. Con eso, nuestro sitio web tendrá seis archivos:

    • readme.md
    • index.html
    • estilos.css
    • capitulo1.html
    • capitulo2.html
    • capitulo3.html

    Archivos markdown

    La redacción de documentos extensos en formato .html puede llegar a ser bastante farragosa por culpa de la presencia de etiquetas. Por este motivo, GitHub permite que redactemos los documentos en formato markdown, y al generar el sitio web, hará la conversión automática a html, añadiendo al texto las etiquetas necesarias.

    Siguiendo con el ejemplo anterior, vamos a borrar el archivo index.html, y crear uno llamado index.md, con el siguiente contenido:

    ---
title: Creando este sitio
author: jf
---

# Mi primera página web

Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed eiusmod
tempor incidunt ut labore et dolore magna aliqua. Ut enim ad minim 
veniam, quis nostrud exercitation ullamco laboris nisi ut aliquid ex ea 
commodi consequat.

Quis aute iure reprehenderit in voluptate velit esse cillum dolore eu 
fugiat nulla pariatur. Excepteur sint obcaecat cupiditat non proident, 
sunt in culpa qui officia deserunt mollit anim id est laborum.

    El formato markdown resulta cómodo de redactar, pero exige respetar algunas normas:

    • Los párrafos deben separarse con una línea en blanco entre ellos.

    • Si queremos un párrafo resaltado como título, debemos ponerle un carácter # como prefijo, con un espacio en blanco separándolo del texto. En la conversión, se sustituirá ese carácter por las correspondientes etiquetas <h1>.

    GitHub cuenta con la posibilidad de establecer “propiedades del documento” en forma de metadatos, que se definen en una sección opcional de cabecera llamada front-matter. Se delimita entre dos líneas formadas por tres guiones cada una. Cada metadato se escribe en formato nombre:valor.

    Nota:

    A diferencia de los formatos HTML y CSS, en los archivos markdown el sangrado a la izquierda del texto tiene valor sintáctico. Se deben respetar unas reglas de escritura que veremos en un capítulo posterior.

    Para generar el sitio web a partir de archivos markdown, GitHub utiliza internamente un software conversor llamado Jekyll. Este programa plantea algunos requerimientos:

    • Tenemos que añadir un archivo de configuración llamado _config.yml (nótese el prefijo en forma de subrayado), que contenga una lista de definiciones de parámetros del proceso de conversión. En su mínima expresión, el contenido de este archivo podría ser:

      title: Mi sitio web

    • Debemos poner nuestra hoja de estilos en una carpeta llamada css

    • Hay que crear un archivo html que sirva como plantilla por defecto para las conversiones. Se llamará default.html y se alojará en la carpeta _layouts.

    En conjunto, nuestra primera página a partir de archivos markdown requerirá lo siguiente:

    • index.md, un documento en formato markdown que se convertirá en index.html.

    • readme.md, un documento con un resumen del proyecto, a efectos de documentación. No es imprescindible.

    • _config.yml, un archivo con la configuración del proceso de conversión, donde cada línea es una pareja parametro:valor.

    • css/estilos.css, un archivo con definiciones de estilos

    • _layouts/default.html será la plantilla en formato html, a utilizar para la conversión de todas las páginas, incluyendo index.html.

    El contenido de la plantilla default.html podría ser algo así como:

    <!DOCTYPE html>
<html>
<head>
    <title>{{ site.title }}</title>

    <link rel="stylesheet" href="css/estilos.css">

</head>
<body>

{{content}}

</body>
</html>

    Las marcas delimitadas entre {{ }} serán sustituidas durante el proceso de conversión por los contenidos.

    La marca {{content}} será sustituida por el texto del archivo markdown, una vez transformado a html.

    La marca {{ site.title }} será sustuida por el parámetro title definido en el archivo de configuración _config.yml.

    Tras hacer todos estos cambios, si activamos el proceso de conversión desde la página settings, el resultado debería ser el mismo que cuando hemos creado el sitio web a partir de archivos .html directamente:

    Nota:

    Los archivos HTML generados por Jekyll no se muestran en el listado de archivos del repositorio.

    Parámetros

    La conversión se puede configurar mediante los parámetros definidos en el front-matter de cada página, o bien en el archivo de configuración del proyecto, _config.yml. La diferencia entre unos y otros es que los primeros afectan a la página en cuestión, y los segundos, a todo el proyecto.

    La forma de influir en los resultados depende del uso que hagamos de ellos en la plantilla. Si por ejemplo, escribimos {{ site.title }} se insertará el título definido en el archivo de configuración. Si escribimos {{ page.title }}, se insertará el título definido en el front-matter de la página.

    Temas

    Un tema es un conjunto de plantillas y hojas de estilo que podemos utilizar como embrión de diferentes proyectos. Cuando creamos un proyecto desde cero, podemos usar un tema creado por nosotros, o bien uno de los que GitHub nos proporciona para contribuir a mejorar el aspecto de nuestro sitio web. Véase la lista de esos temas en esta página. Pulsando sobre cualquiera de ellos, podemos ir al repositorio del mismo, donde descubriremos una colección de archivos de plantilla y estilos que GitHub Pages tomará para unirlos a nuestros documentos y formar la página web definitiva.

    No tenemos por que limitarnos a lo proporcionado por GitHub; existen cientos de temas de terceros disponibles en la red. Véase una amplia colección en páginas como esta o en esta otra. En realidad, el uso de esos temas no se limita a GitHub, sino que podemos utilizarlos allá donde usemos Jekyll para generar páginas web.

    Generalmente el procedimiento para usar un tema es copiar todo su contenido en nuestro repositorio. En el caso de muchos de ellos (y en particular los proporcionados por GitHub), bastará con hacer mención al tema que queremos utilizar en el archivo de configuración _config.yml. Por ejemplo:

    title: Mi página web
theme: jekyll-theme-cayman

    y en este caso el proceso de conversión tomará automáticamente los archivos ubicados en el repositorio del tema en cuestión.

    Aplicando un tema

    Para crear un proyecto con un tema de los proporcionados por GitHub, vamos a comenzar con un proyecto en blanco. Entramos en la página settings, apartado GitHub Pages, seleccionamos la rama master, y al pulsar en el botón [Choose a Theme] se mostrará una lista de plantillas disponibles. Seleccionamos una de ellas y pulsamos en el botón [Select Theme]

    Se generá automáticamente un nuevo archivo readme.md con un texto “de pruebas” en formato markdown:

    Pulsamos en el botón Commit para guardarlo.

    GitHub creará un archivo index.html a partir de readme.md o de index.md. Si se encuentra con los dos, ignorará readme.md.

    Vamos a cambiar el nombre de readme.md por index.md. Esta será nuestra página principal, que GitHub convertirá a index.html.

    Véase que también se ha creado automáticamente el archivo _config.yml con las variables de configuración del proyecto. Lo modificaremos para adaptar la página a nuestro gusto. Comencemos por añadir las variables title y description, que será lo que este tema muestra en la cabecera de la página.

    theme: jekyll-theme-cayman
title: Mi página personal
description: mi primera página

    El resultado final es el siguiente:

    Finalmente, nuestro primer proyecto tiene por ahora dos archivos creados de forma automática y uno que crearemos manualmente:

    • index.md - Archivo de texto en formato markdown con el texto de nuestra primera página. La extensión de los ficheros puede ser “.md” o “.markdown”.

    • _config.yml - Archivo de texto en formato yml. Contiene las variables de configuración del proyecto.

    • readme.md- Una descripción del proyecto en formato markdown. Se mostrará al visitar el repositorio GitHub.

    El resto de archivos se toman automáticamente del repositorio del tema, con la excepción de aquellos que creemos en nuestro repositorio con el mismo nombre y carpeta, que tendrán prioridad.

    Acerca de Jekyll

    GitHub Pages utiliza como software generador de HTML el mencionado programa Jekyll, cuyo manual de uso está en su página web. En las plantillas se utiliza como complemento del texto HTML el lenguaje de marcas {{...}} Liquid cuya referencia podemos encontrar aquí.

    Sin embargo, Jekyll no es privativo de GitHub Pages. Un ejemplo de uso lo tenemos en Cloudcannon, un alojamiento web comercial que también lo utiliza. Tiene una fantástica colección de tutoriales para aprender a utilizarlo.

    Otro ejemplo digno de ser examinado es The Programming Historian, un sitio web dedicado al alojamiento de manuales técnicos. En esta página tenemos uno que explica como utilizar GitHub Pages. De hecho, este sitio también está creado usando GitHUb Pages y Jekyll. Explorando su repositorio de documentos podemos aprender bastante acerca de todo el proceso.
