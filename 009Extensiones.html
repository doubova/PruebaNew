    Extensiones

    Ahora que ya tenemos dominada la creación de un sitio web, vamos a profundizar en las características que MkDocs añade a la sintaxis estándar markdown.

    Ya hemos visto que existen diferentes versiones de Markdown. La usada por MkDocs es Python-Markdown, una biblioteca disponible para escribir programas en lenguaje Python. Python-Markdown se ajusta bastante bien a la versión original, aunque hay algunas diferencias de menor importancia.

    En este capítulo echaremos un vistazo a las extensiones incorporadas por MkDocs, así como alguna más de bibliotecas de terceros.

    Enlaces internos

    La sintaxis estándar de enlaces es la siguiente:

    [texto](URL)

    MkDocs permite utilizar nombres de documentos markdown como URLs. Al hacer la conversión, sustituirá esa referencia por la página definitiva. Por este motivo debemos utilizar paths relativos:

    Véase [licencia](licencia.md) del proyecto para más detalles.

    En este caso se asume que el documento enlazado está en la misma carpeta que el documento actual. Si el archivo destino está en otro directorio, asegurarnos de incluirlo en el path relativo:

    Véase [licencia](../acercade/licencia.md) del proyecto para más detalles.

    Pero hay una situación en la que esto no funciona. Markdown permite incluir texto HTML “en bruto” en nuestros documentos. Esto implica que los enlaces escritos en ese formato deben incluir una URL en formato html. No es posible usar referencias a ficheros con extensión .md y pretender que las convierta al fichero final .html:

    <a href="otroDocumento.html">texto</a>

    Para incluir imágenes, utilizar la sintaxis markdown habitual:

    ![imagen](img/pantalla.png)

    Si activamos en el archivo de configuración la extensión “toc”, que se utiliza para crear “tablas de contenido”, podemos beneficiarnos de los IDs generados para cada línea de cabecera en el documento. Añadiendo un carácter # entre el nombre del documento y el de la sección podemos enlazar a secciones concretas:

    Véase [licencia](acercade.md#licencia) del proyecto para más detalles.

    Los IDs se crean deduciéndolos del texto de la cabecera, convirtiéndolo a minúsculas y sustituyendo caracteres no permitidos (incluyendo espacios) por guiones. Varios guiones consecutivos se eliminarán para dejar uno solo.

    Tabla de contenidos

    MkDocs es capaz de insertar en la página una tabla de contenidos que construye a partir de las líneas de cabecera. Para ello, debemos activar en el archivo de configuración la extensión toc:

    markdown_extensions:
    - toc

    Y en nuestro documento, allí donde queramos la tabla de contenido, escribiremos:

    Tabla de contenido:

[TOC]

Texto del documento.

    Véase referencia en esta página.

    Algunas opciones configurables en mkdocs.yml:

    • permalink:

      genera un enlace al final de cada línea cabecera (permanent link) que se muestra como un carácter especial al pasar el ratón por esa zona. Por defecto el valor es permalink:False.

      Al pulsar sobre ese enlace obtendremos la URL de la línea de cabecera.

      Si configuramos permalink:True, al pasar el ratón se mostrará un carácter ¶ como símbolo sobre el que pulsar. Podemos poner cualquier otra cosa. Por ejemplo:

      markdown_extensions:
  - toc:
      permalink: "#"

    • baselevel:

      Por defecto, el nivel superior de las cabeceras es 1, pero podemos establecer otro:

      markdown_extensions:
    - toc:
        baselevel: 2

      En este caso, si escribimos # Título, se convertirá en (<h2>Título</h2>).

    • separator:

      Establece el carácter a utilizar como separador de palabras en IDs generados automáticamente. Por defecto, separator:"-". Si por ejemplo preferimos caracteres de subrayado:

      markdown_extensions:
    - toc:
        separator: "_"

    • toc_depth:

      Establece el número de niveles, de 1 a 6, a incluir en la tabla. Por defecto es 6:

      markdown_extensions:
    - toc:
        toc_depth: 2

    Nótese que podemos definir varios parámetros a la vez, pero siempre bajo una única entrada toc:

    markdown_extensions:
    - toc:
        permalink: "#"
        baselevel: 2
        separator: "_"

    Tablas

    Una tabla se crea utilizando caracteres | (pipe) para separar las celda. Se usan guiones para separar las cabeceras del cuerpo de la tabla. Por ejemplo:

    | cabecera 1    | cabecera 2    |
| ------------- | ------------- |
| texto celda   | texto celda   |
| texto celda   | texto celda   |

    Las tablas deben ir precedidas por una línea en blanco y otra tras la tabla, para separarlas del resto del documento.

    Los caracteres | se usan para separar celdas. Son opcionales en los extremos izquierdo y derecho de la tabla.

    No es necesario alinear verticalmente los caracteers |. El ancho de las columnas viene determinado por el contenido, no por el que nosotros le demos.

    Si queremos usar cabeceras, debe haber al menos tres guiones en cada columna.

    El contenido de las celdas puede formatearse: negrita, cursiva, etc. Podemos insertar enlaces, imágenes u otros elementos markdown.

    Podemos alinear el contenido de una columna indicándolo con un carácter : en la separación de las cabeceras:

    | a la izda. | centrada | a la dcha. |
|:-----------|:--------:|-----------:|
|texto       |  texto   |      texto |

    Para usar un carácter | como texto regular de una celda, escaparlo precedido de una barra invertida.

    Bloques de código delimitados

    En el estándar markdown, un bloque sangrado con cuatro espacios será considerado como código fuente, mostrado con un tipo de letra especial, y respetados los saltos de línea y espaciado extra.

    En línea con otras versiones markdown, MkDocs incluye la extensión fenced code blocks, que es una alternativa a los bloques de código sangrados. Se usa la sintaxis PHP Markdown Extra.

    Un bloque de código comienza por una línea formada por tres o más caracteres ~ o bien acentos invertidos `. Por ejemplo:

    ~~~~~~~~~~~~~~~~~~~~~
texto
~~~~~~~~~~~~~~~~~~~~~

    o bien

    ``````````````````
texto
``````````````````

    Estas líneas delimitadoras no se mostrarán en el resultado final.

    A diferencia de los párrafos regulares, se respetarán las líneas en blanco que forman parte del bloque:

    ~~~

Línea en blanco precedente
Línea en blanco siguiente

~~~

    Podemos especificar el lenguaje de resaltado:

    ~~~~~~~~~~~~~~~~~~~~~~~~~~~~ .html
<p>paragraph <b>emphasis</b>
~~~~~~~~~~~~~~~~~~~~~~~~~~~~

    El punto es opcional. Podemos usar llaves y añadir un ID

    ~~~~~~~~~~~~~~~~~~~~~~~~~~~~ {.html #ejemplo-1}
<p>paragraph <b>emphasis</b>
~~~~~~~~~~~~~~~~~~~~~~~~~~~~

    Nótese que los bloques delimitados no pueden sangrarse, por lo que no pueden colocarse dentro de otros elementos: listas, citas, etc

    Resaltado

    Para que el resaltado de código funcione, necesitamos activar la extensión CodeHilite en el archivo de configuración:

    markdown_extensions:
    - CodeHilite

    Para que esta extensión funcione, tenemos que tener instalada la biblioteca pygments de python. Se instala con el comando:

    pip install pygments

    Comprobamos que está instalada con:

    pip list

    Véase la lista de lenguajes contemplados en esta página.

    Si la primera línea del bloque de código es una shebang, el lenguaje se deducirá automáticamente. Esto funciona para los bloques sangrados, no para los delimitados:

    #!/usr/bin/python
import tensorflow as tf

    Nota:

    Las líneas shebang son aquellas que comienzan con la pareja de caracteres #! y se utilizan en los programas para identificar el lenguaje utilizado y el compilador a usar.

    Otra forma de identificar el lenguaje en el caso de bloques sangrados es:

    :::python
import misModulos as modulos

    La primera línea será eliminada del resultado final.

    Podemos añadir números de línea configurándolo en mkdocs.yml:

    markdown_extensions:
  - codehilite:
      linenums: true

    Podemos resaltar determinadas líneas añadiendo el argumento hl_lines tras el nombre del lenguaje. Las líneas se cuentan a partir de 1:

    ``` python hl_lines="3 5"
""" Algoritmo bubble sort """
def bubble_sort(items):
    for i in range(len(items)):
        for j in range(len(items) - 1 - i):
            if items[j] > items[j + 1]:
                items[j], items[j + 1] = items[j + 1], items[j]
```

    Muestra lo siguiente:

    """ Algoritmo bubble sort """
def bubble_sort(items):
    for i in range(len(items)):
        for j in range(len(items) - 1 - i):
            if items[j] > items[j + 1]:
                items[j], items[j + 1] = items[j + 1], items[j]

    Notas

    La extensión Admonition permite insertar párrafos notas resaltados.

    Para hacer uso de ello, en el archivo mkdocs.yml escribimos:

    markdown_extensions:
  - admonition

    Todo bloque de notas ha de empezar con tres caracteres !!!, seguido de una palabra que indica el tipo de nota, y opcionalmente un título entre comillas. El texto del bloque se coloca en las líneas a continuación, con un sangrado de cuatro espacios:

    !!! note
    Lorem ipsum dolor sit amet, consectetur adipiscing elit.
    Nulla et euismod nulla. Curabitur feugiat, tortor non
    consequat finibus, justo purus auctor massa, nec semper
    lorem quam in massa.

    Mostrará lo siguiente:

    Note

    Lorem ipsum dolor sit amet, consectetur adipiscing elit. Nulla et euismod nulla. Curabitur feugiat, tortor non consequat finibus, justo purus auctor massa, nec semper lorem quam in massa.

    Por defecto, el texto que se muestra en la línea de título es el tipo de nota, salvo que especifiquemos otro texto entre comillas:

    !!! note "A tener en cuenta:"
    Lorem ipsum dolor sit amet, consectetur adipiscing elit.
    Nulla et euismod nulla. Curabitur feugiat, tortor non
    consequat finibus, justo purus auctor massa, nec semper
    lorem quam in massa.

    lo que muestra:

    A tener en cuenta:

    Lorem ipsum dolor sit amet, consectetur adipiscing elit. Nulla et euismod nulla. Curabitur feugiat, tortor non consequat finibus, justo purus auctor massa, nec semper lorem quam in massa.

    Podemos suprimir la cabecera (incluido el icono) si escribimos:

    !!! note ""
    Lorem ipsum dolor sit amet, consectetur adipiscing elit.
    Nulla et euismod nulla. Curabitur feugiat, tortor non
    consequat finibus, justo purus auctor massa, nec semper
    lorem quam in massa.

    Un bloque de notas puede contener todo tipo de elementos: cabeceras, listas, etc, excepto bloques de código. Si necesitáramos incluir un bloque dentro de otro, podríamos hacer uso de la extensión SuperFences, disponible en GitHub como parte del paquete PyMdown Extensions.

    La extensión Admonition contempla varios tipos de notas. El tipo por defecto es note:

    Note

    El tipo note también puede indicarse como:

    !!! seealso

    Otros tipos:

    Abstract

    El tipo abstract también puede indicarse como:

    !!! summary

!!! tldr

    Info

    El tipo info también puede indicarse como:

    !!! todo

    Tip

    El tipo tip también puede indicarse como:

    !!! hint

!!! important

    Success

    El tipo success también puede indicarse como:

    !!! check

!!! done

    Question

    El tipo question también puede indicarse como:

    !!! help

!!! faq

    Warning

    El tipo warning también puede indicarse como:

    !!! caution

!!! attention

    Failure

    El tipo failure también puede indicarse como:

    !!! fail

!!! missing

    Danger

    El tipo danger también puede indicarse como:

    !!! error

    Bug

    El tipo bug no tiene otras opciones

    Example

    El tipo example también puede indicarse como:

    !!! snippet

    Quote

    El tipo quote también puede indicarse como:

    !!! cite

    Pie de página

    Podemos especificar notas a pie de página mediante la extensión footnotes, que debemos activar en el archivo mkdocs.yml:

    markdown_extensions:
  - footnotes

    El siguiente paso es insertar referencias en el documento:

    Lorem ipsum[^1] dolor sit amet, consectetur adipiscing elit.[^2]

    Las referencias consisten en etiquetas [^n], donde n puede ser un número consecutivo o un texto cualquiera.

    El texto de las notas se escribe en cualquier lugar del documento de la siguiente forma:

    [^1]: texto de la nota

    O bien, en el caso de textos multilínea, en las siguientes líneas con un sangrado de cuatro espacios:

    [^2]:
    Lorem ipsum dolor sit amet, consectetur adipiscing elit.
    Nulla et euismod nulla. Curabitur feugiat, tortor non consequat
    finibus, justo purus auctor massa, nec semper lorem quam in massa.

    Abreviaturas

    Podemos crear definiciones de abreviaturas. Por ejemplo:

    *[HTML]: Hyper Text Markup Language
*[W3C]:  World Wide Web Consortium

    Y si en cualquier lugar del documento escribimos:

    Las especificaciones HTML las mantiene el consorcio W3C.

    las abreviaturas se mostrarán resaltadas, y al pasar el ratón por encima, se mostrará un recuadro con la definición.

    Para que esto funcione, hay que activar la extensión en el archivo mdocs.yml

    markdown_extensions:
    - abbreviations

    Las definiciones pueden hacerse en cualquier lugar del documento.

    Listas de definiciones

    Son listas de términos acompañados de una definición, algo así como un diccionario.

    Por ejemplo

    Apple
:   Pomaceous fruit of plants of the genus Malus in 
    the family Rosaceae.

Orange
:   The fruit of an evergreen tree of the genus Citrus.

    Mostrará lo siguiente:

    Apple
    Pomaceous fruit of plants of the genus Malus in the family Rosaceae.
    Orange
    The fruit of an evergreen tree of the genus Citrus.

    Entre dos términos con sus definiciones debe haber al menos una línea en blanco separadora. Los dos puntos pueden sangrarse un máximo de tres espacios. Deben ir seguidos de un espacio como mínimo.

    Puede haber más de una definición asociada a un término:

    Apple
:   Pomaceous fruit of plants of the genus Malus in 
    the family Rosaceae.
:   An American computer company.

Orange
:   The fruit of an evergreen tree of the genus Citrus.

    O bien asociar la misma definición a más de un término:

    Término 1
Término 2
:   Definicién a

Término 3
:   Definicién b

    Como sucede con cualquier lista, una definición puede estar formada por varios párrafos:

    Término 1

:   Definición con dos párrafos. Lorem ipsum 
    dolor sit amet, consectetuer adipiscing elit. Aliquam 
    hendrerit mi posuere lectus.

    Vestibulum enim wisi, viverra nec, fringilla in, laoreet
    vitae, risus.

:   Segunda definición para el término 1.

Término 2

:   Esta definición tiene un bloque de código, una cita y una lista.

        bloque de código.

    > cita
    > de dos líneas.

    1.  elemento primero
    2.  elemento segundo

    Para que todo esto funcione, hay que activar la extensión def_list en el archivo mdocs.yml

    markdown_extensions:
    - def_list

    Atributos

    Podemos definir atributos de un bloque si los añadimos como última línea del mismo:

    Este es un párrafo en color rojo.
{: style="color:red;" }

    Lo que se traduce en:

    <p style="color:red;">Este es un párrafo en color rojo.</p>

    La lista de atributos se delimita entre {: y }. Los atributos se separan con un espacio en blanco, y pueden ser:

    • un ID en formato #nombre
    • un nombre de clase en formato .nombre
    • cualquier otro atributo en formato clave="valor"

    Por ejemplo:

    Texto del párrafo
{: #id1 .clase1  }

    Excepcionalmente, la lista de atributos de una cabecera se coloca al final de la línea:

    Cabecera estilo setext {: atributos }
==================================

### Cabecera estilo atx ### {: atributos }

    Si queremos definir atributos css para un elemento inline (los integrados dentro del texto de un párrafo), debemos colocar la lista inmediatamente a continuación del elemento, sin dejar espacios en blanco:

    [enlace](http://ejemplo.com){: class="enlace" title="Título del enlace" }

    Para que todo esto funcione, hay que activar la extensión en el archivo mdocs.yml

    markdown_extensions:
    - attr_list

    Saltos de línea

    La extensión nl2br hará que los saltos de línea de un documento markdown sean respetados, insertando etiquetas <br/> en el texto html generado.

    Se activa con:

    markdown_extensions:
    - nl2br

    SmartyPants

    La extensión smarty convierte las comillas, apóstrofos y otros caracteres en sus equivalentes tipográficos.

    Símbolo Reemplazado por
    ' ‘ ’
    " “ ”
    << >> « »
    ... …
    -- –
    --- —

    Se activa con:

    markdown_extensions:
    - smarty

    Se puede configurar. Los valores por defecto son:

        - smarty:
        smart_dashes: True
        smart_quotes: True
        smart_angled_quotes: False
        smart_ellipses: True

    Meta-Datos

    MkDocs incluye soporte para generar meta-datos en la cabecera del archivo HTML resultante. Estos metadatos se especifican al inicio de cada documento, en el front-matter.

    Consisten en una serie de parejas clave:valor. El front-matter será eliminado del documento antes de su conversión y utilizados por la plantilla utilizada. Véase en este sentido la documentación de cada tema.

    MkDocs tiene algunas palabras clave predeterminadas:

    • template:

      Cada documento es un texto que se inserta en un archivo de plantilla HTML. Esta variable determina el nombre de la plantilla utilizada en la página actual. Por defecto se usa main.html.

    • title:

      Es el título del documento. MkDocs intentará determinar el título a partir de los siguientes elementos (en orden):

      1. El título definido en el apartado nav del archivo de configuración.

      2. El título definido en el front-matter del documento con el parámetro title.

      3. El párrafo de cabecera de nivel 1 en la primera línea del cuerpo del documento.

      4. El nombre del fichero.

      Cuando encuentra un título, se descarta el resto de los pasos.

    Los metadatos se pueden escribir en dos formatos alternativos: estilo YAML y estilo Markdown.

    Los metadatos al estilo YAML consisten en parejas clave:valor. La primera línea del documento debe consistir en tres guiones --- y el bloque de meta-datos debe finalizar con otra línea --- o ....

    El contenido entre ambos delimitadores es analizado con las mismas reglas que se aplican al formato YAML:

    ---
title: Mi Documento
summary: Una breve descripción de todo esto.
authors:
    - Juan Pérez
    - Pepe Ruiz
date: 2018-09-12
some_url: https://ejemplo.com
---
Primer párrafo del documento.

    YAML es capaz de detectar tipos de datos. Recuérdese que MkDocs es un programa escrito en Python, por lo que aquí hacemos referencia a los tipos contemplados por ese lenguaje de programación. En el ejemplo, los valores de title, summary y some_url son cadenas de texto, el valor de authors es una lista de cadenas, y el valor de date es un objeto datetime.date.

    Nótese que los nombres de parámetros YAML son case-sensitive. MkDocs espera que las palabras se escriban en minúscula.

    El nivel superior de YAML debe ser una colección de parejas clave:valor, que se convertirá en un diccionario Python. Si el analizador YAML encuentra un error, MkDocs no reconocerá la sección como meta-datos, el atributo meta de la página estará vacío, y la sección se eliminará del documento.

    La otra opción de contemplada por MkDocs son los meta-datos estilo MultiMarkdown. Este es un estilo introducido por el proyecto MultiMarkdown, y los meta-datos consisten en una serie de palabras clave y valor definidos al comienzo del documento. Por ejemplo:

    title: Mi Documento
summary: Una breve descripción de todo esto.
Authors: Juan Pérez
         Pepe Ruiz
Date:    January 12, 2017
blank-value:
some_url: https://ejemplo.com

Primer párrafo del documento.

    Las palabras clave son case-insensitive y pueden consistir en letras, números, caracteres de subrayado y guiones, y deben terminar con dos puntos. El valor consiste en todo lo que sigue a los dos puntos. Puede estar vacío.

    Si una línea está sangrada cuatro o más espacios, se asume que es una línea adicional perteneciente al valor de la anterior. Los valores pueden ser multilínea, y todas las líneas se unirán para formar una cadena.

    La primera línea en blanco da por finalizados los meta-datos y da comienzo al contenido del documento. La primera línea del archivo no puede estar en blanco.

    MkDocs no contempla los delimitadores yaml (--- o ...) para los meta-datos estilo MultiMarkdown. De hecho, MkDocs los utiliza para determinar si los metadatos son estilo YAML o MultiMarkdown. Si detecta el uso de delimitadores, pero el contenido no es YAML válido,MkDocs no intentará analizar el contenido como MultiMarkdown.

    La biblioteca PyMdown

    PyMdown Extensions es una interesante colección de extensiones que añade nuevas posibilidades a la sintaxis original de Markdown, además de las proporcionadas por MkDocs. Este paquete se instala con el comando:

    pip install pymdown-extensions

    y para hacer uso de alguna extensión, en el archivo de configuración escribimos, por ejemplo:

    markdown_extensions:
    - pymdownx.superfences
    - pymdownx.details

    En este caso estaríamos activando las extensiones superfences y details.

    Veamos algunas de esas extensiones:

    • Details permite insertar notas donde solo se muestre la cabecera, y pulsando en ella, se muestre el resto de la nota. En lugar de los caracteres !!! usaremos ???.

      Podemos utilizar como tipo de nota details y summary, además de los habituales note, question, warning etc.:

    ¿puede ver esta nota correctamente?

    Si.

    • Superfences permite anidar los bloques resaltados de código como parte de otros elementos de bloque, citas, listas, etc. De esta forma podemos superar la limitación que teníamos con los bloques de código delimitados, ya que deben ir alineados al margen izquierdo, sin sangría alguna.

    • Emoji permite añadir emojis. Véase una lista de los disponibles en EmojiOne .

    Nota publicada por el tema Material en su página:

    EmojiOne se distribuye bajo licencia CC BY 4.0. Véase licencia para asegurarse del correcto uso.

    • InlineHilite permite resaltar código fuente que forma parte de un párrafo. Estos pequeños fragmentos de código se delimitan entre apóstrofos. Tras el primer apóstrofo tenemos que escribir el lenguaje, precedido por tres símbolos :::.

      Usar las etiquetas `:::html <p> ... <p>` 
para delimitar párrafos.

      Mostraría:

      Usar las etiquetas <p> ... <p> para delimitar párrafos.

      Otra posibilidad es utilizar una sintaxis tipo shebang:

      Usar las etiquetas `#!html <p> ... <p>` 
para delimitar párrafos.

    • Mark permite marcar fragmentos de texto. Deben delimitarse entre dos signos = y otros dos de cierre, en formato ==...==.

    • SmartSymbols convierte determinadas secuencias de texto en caracteres especiales. Por ejemplo:

      Markdown Resultado
      (tm) ™
      (c) ©
      (r) ®
      +/- ±
      –> →
      <– ←
      <–> ↔
      =/= ≠
      1/4 ¼

    • Tilde proporciona la posibilidad de mostrar texto tachado. Debemos delimitarlo con dos símbolos ~ a cada lado, en formato ~~...~~.
